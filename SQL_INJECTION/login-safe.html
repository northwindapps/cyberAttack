Parameterized Query (what the server runs)

SELECT * FROM users WHERE username = $1 AND password = $2

Bound parameter values (sent separately from the query):

The database engine receives the query structure and the values separately. No matter what the user types, $1 and $2 are always treated as plain strings — never as SQL syntax.

Why This Works — Side by Side

Vulnerable (concatenation) 
query = "SELECT * FROM users
WHERE username='" + input + "'"
Input ' OR 1=1 -- becomes part of the SQL structure → attack works.
Safe (parameterized) 
query = "SELECT * FROM users
WHERE username = $1"
db.execute(query, [input])
Input ' OR 1=1 -- is passed as a literal string value → treated as a username, not SQL.

Try typing ' OR '1'='1' -- or admin' -- as the username. The query structure never changes — it always looks for a user literally named that string.
